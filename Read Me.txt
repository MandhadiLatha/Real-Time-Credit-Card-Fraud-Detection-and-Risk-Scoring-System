## Project Overview

Credit card fraud causes significant financial losses for banks and payment processors every year. Since fraudulent transactions represent only a tiny fraction of total transactions, traditional rule-based systems often produce excessive false alarms while missing sophisticated fraud patterns.

This project develops an end-to-end machine learning pipeline to identify fraudulent credit card transactions using historical transaction data. In addition to fraud classification, the project generates a **Fraud Risk Score** that can be used for transaction prioritization and operational monitoring.Tableau dashboard provides business stakeholders with insights into fraud trends, transaction behavior, model outputs, and fraud risk distribution.


# Business Problem

Financial institutions need to:
* Detect fraudulent transactions quickly
* Reduce financial losses
* Minimize false positives
* Prioritize high-risk transactions for investigation

### Key Findings

* Fraud transactions account for approximately **0.17%** of total transactions.
* The dataset is extremely imbalanced.
* Fraudulent transactions are generally concentrated in lower transaction value ranges.
* Transaction amount alone is not a reliable predictor.
* PCA-transformed variables prevent direct business interpretation.

# Handling Class Imbalance
Since fraud cases are extremely rare, the project applies techniques to address class imbalance before model training.
Evaluation focuses on metrics that are more informative than overall accuracy.

# Machine Learning Models

The following supervised learning algorithms were evaluated:
* Logistic Regression
* Random Forest
* XGBoost

# Evaluation Metrics

Because the dataset is highly imbalanced, the following metrics were used:
* Precision
* Recall
* F1 Score
* ROC-AUC
* Precision-Recall AUC
* Confusion Matrix

Accuracy was not used as the primary evaluation metric because it can be misleading for highly imbalanced datasets.

# Fraud Risk Scoring

Instead of only predicting whether a transaction is fraudulent, the model generates a **Fraud Risk Score** using predicted probabilities.
Example:

| Probability | Risk Level  |
| ----------- | ----------- |
| 0.00 – 0.30 | Low Risk    |
| 0.30 – 0.70 | Medium Risk |
| 0.70 – 1.00 | High Risk   |

This allows fraud analysts to prioritize investigations based on estimated fraud likelihood.
